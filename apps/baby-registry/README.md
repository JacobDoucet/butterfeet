# üåø Baby Registry

A multi-tenant baby registry web app. Each registry lives at its own subpath
(`/r/<slug>`), gift-givers can claim items publicly with an optional message,
and the registry owner manages everything from `/owner`.

- **Backend** ‚Äî Go, MongoDB, with model code generated by
  [forge](https://github.com/JacobDoucet/forge) from YAML specs.
- **Frontend** ‚Äî Vite + React + TypeScript + MUI.
- **Auth** ‚Äî passwordless magic-link for owners. The giver flow is fully public.
- **Item import** ‚Äî paste any product URL and we extract title/image/price via
  OpenGraph. Built-in detection for Amazon, Mamas & Papas, Etsy, John Lewis, IKEA.
- **Deploy** ‚Äî `docker compose up --build`.

## Quickstart

```sh
# Bring everything up
cd apps/baby-registry
docker compose up --build

# Open
open http://localhost:3035        # frontend (SPA)
# Backend API is proxied through nginx; reachable directly at :3036.
```

Magic links are logged to the backend container's stdout in dev. Grep for
`MAGIC LINK`:

```sh
docker compose logs -f backend | grep MAGIC
```

## Local development

```sh
# 1) Install forge once
go install github.com/JacobDoucet/forge@v1.0.5

# 2) Generate code from the YAML spec
cd apps/baby-registry
forge build

# 3) Start Mongo (or `docker compose up mongo`)
# 4) Run the backend
cd apps/backend
MONGO_URI=mongodb://localhost:27017 \
  DB_NAME=baby_registry \
  JWT_SECRET=dev-secret \
  APP_BASE_URL=http://localhost:5173 \
  go run .

# 5) Run the frontend
cd ../frontend
npm install
npm run dev
# ‚Üí http://localhost:5173 (Vite proxies /api to :8080)
```

## Editing the data model

Edit YAML under `models/` and re-run `forge build`. Forge regenerates Go models,
MongoDB persistence, HTTP handlers, and a TypeScript client.

> **Note**: there is one manual file in `apps/backend/generated/owner_user/actor_extra.go`
> that adds a `GetOwnerId()` method required by the generated `permissions.Actor`
> interface. Keep this file around between regenerations.
